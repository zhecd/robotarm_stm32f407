# 上位机通信协议接口文档

## —— SCARA 教学机械臂 UART G-code 协议

---

## 一、物理层

| 参数 | 值 |
|------|-----|
| 接口 | UART1 (PA9=TX, PA10=RX) |
| 波特率 | 115200 bps |
| 数据位 | 8 |
| 校验位 | 无 |
| 停止位 | 1 |
| 电平 | 3.3V TTL (如需 USB 连接请用 USB-TTL 模块) |
| 缓冲区 | 256 字节环形缓冲 |
| 行超时 | 50ms (一行内字符间隔超过 50ms 视为超时) |

---

## 二、协议概述

**文本行协议，一问一答。** 上位机发送一行 G-code 指令，机械臂**阻塞执行完毕后**回复一行确认。

```
上位机 → 机械臂:  "G1 X100 Y200 Z150 F3000\r\n"
机械臂 → 上位机:  "ok\r\n"          ← 运动完成后才回复
```

特点：
- 一条指令完成前，机械臂不接收下一条（阻塞执行）
- 复合指令可以混合参数：`G1 X100 Y200 Z150 F3000`
- 参数顺序无关：`G1 F3000 X100` 和 `G1 X100 F3000` 等价
- 指令大小写不敏感：`g1`, `G1`, `m3`, `M3` 均可
- 未知指令会被忽略，不回复

---

## 三、支持的指令

### 3.1 G0 / G1 —— 直线运动到目标点

```
G0  X<mm> Y<mm> Z<mm> F<mm/min>
G1  X<mm> Y<mm> Z<mm> F<mm/min>
```

在本固件中，**G0 和 G1 完全等价**，均执行 quintic smoothstep 平滑直线插补。

| 参数 | 含义 | 单位 | 可选 | 默认值 |
|------|------|------|------|--------|
| X | 目标 X 坐标 | mm | 是 | 保持当前位置 |
| Y | 目标 Y 坐标 | mm | 是 | 保持当前位置 |
| Z | 目标 Z 坐标 | mm | 是 | 保持当前位置 |
| F | 进给速度 | mm/min | 是 | 3000 (首次) / 上次设定值 |

- 未指定的轴保持原位
- F 参数具有记忆效应——设定后会一直沿用，直到再次修改
- 距离 < 0.1mm 的移动会被跳过（不产生运动帧）

**回复**：`ok\r\n`

**示例**：
```
G1 X150 Y200 F2000        → 移动到 X=150, Y=200, Z 不动, 速度 2000mm/min
G1 Z100                   → Z 轴从当前位置降到 100, 保持上次 F 值
G1 X0 Y185 Z240 F3000     → 回到初始位置
```

### 3.2 M3 —— 打开夹爪

```
M3
```

无参数。夹爪舵机转到预设的张开角度。

**回复**：`M3OK\r\n`

### 3.3 M5 —— 关闭夹爪

```
M5
```

无参数。夹爪舵机转到预设的闭合角度。

**回复**：`M5OK\r\n`

## 四、错误处理

解析失败时回复：

```
error: Parse failed!\r\n
```

常见原因：
- 行中不含任何有效指令（无 G0/G1/M3/M5）
- 缓冲区溢出
- UART 线路噪声（伺服 PWM 可能干扰 PA10，导致收到乱码字节 0x0D）

上位机建议实现超时重试：发指令后 5 秒内未收到回复，重发一次。

## 五、坐标系

```
Z+ (向上)
 │
 │  桌面 Z=0
 │
 ├────────────────────────────── Y+
 │
 │   机械臂底座在 (0, 0, 140)
 │   初始位置 (归零后)：X=0, Y=185, Z=240
 │
 └────────────────────────────── X+
```

**坐标原点**：底座投影到桌面的位置，Z=0 为桌面高度。

**机械参数**：

| 参数 | 值 | 说明 |
|------|-----|------|
| 底座高度 | 140mm | 桌面到肩关节转轴 |
| 连杆 1 | 140mm | 肩到肘 |
| 连杆 2 | 140mm | 肘到腕 |
| 工具水平偏移 | 45mm | 腕到夹爪中心 |
| 工具垂直偏移 | -40mm | 腕到夹爪尖端 |
| 理论最大半径 | ~235mm | 从底座中心算起 |

**注意**：上位机在发送坐标前应做软限位检查。超出臂展范围的目标坐标虽然不会撞机（运动学会将其 clamp 到最大半径），但会导致预期外的位置偏差。

## 六、指令时序

```
上位机发送              │  机械臂状态            │  机械臂回复
────────────────────────┼───────────────────────┼─────────────
"G1 X150 Y200\r\n"      │  开始运动...          │
                        │  (阻塞, 不接收)        │
                        │  运动完成              │
                        │  误差补偿中...         │
                        │  补偿完成              │
                        │  PID 目标同步          │
                        │                       │  "ok\r\n"
"M3\r\n"                │  夹爪打开              │  "M3OK\r\n"
"G1 X100 Z150\r\n"      │  移动到拾取位置        │  "ok\r\n"
"M5\r\n"                │  夹爪闭合 (抓取)        │  "M5OK\r\n"
```

**关键时序约束**：
1. 必须等收到上一条回复后才能发下一条
2. 运动包含 4 个阶段：直线插补 → 误差补偿（迭代修正到编码器目标）→ 100ms 稳定延时 → PID 闭环同步
3. 总耗时 ≈ 运动时间 + 补偿时间（通常 < 1s）+ 100ms 延时

## 七、典型拾放流程

```
1. G1 X120 Y180 Z80 F2000     ← 移动到抓取位置上方
2. G1 Z50                     ← 下降
3. M5                         ← 关闭夹爪 (抓取)
4. G1 Z120                    ← 抬升
5. G1 X200 Y150 Z120          ← 移动到放置位置上方
6. G1 Z50                     ← 下降
7. M3                         ← 打开夹爪 (释放)
8. G1 Z150                    ← 抬升
9. G1 X0 Y185 Z240            ← 回到初始位置
```

- [ ] 软限位校验（半径 ≤ 235mm, Z ≥ 0）
- [ ] 收到 `M3OK` 后夹爪状态记为"开"，`M5OK` 后记为"关"

- [ ] `error: Parse` 时记录日志并重试
