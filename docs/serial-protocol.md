# UART 与 G-code 串口协议

本文档定义 `robotarm_stm32f407` 与上位机之间的 UART 文本协议。上位机应以本文件为唯一协议依据，不应根据串口日志推测命令结束条件或运动完成状态。

## 1. 物理与传输参数

| 项目 | 参数 |
|---|---|
| 接口 | UART1，PA9 为 TX、PA10 为 RX |
| 波特率 | 115200 bit/s |
| 格式 | 8N1，无校验、1 个停止位 |
| 电平 | 3.3 V TTL；连接 USB 时使用 USB-TTL 转换器 |
| 接收 | DMA 循环缓冲 |
| 发送 | 非阻塞中断发送队列 |
| 最大命令行 | 255 个可用字符 |
| 命令结束符 | **CRLF**，即 `\r\n` |

命令必须使用文本模式发送，并在每条命令末尾附加真实的回车和换行字节 `0x0D 0x0A`。发送文本 `\\r\\n` 本身不会产生结束符。

## 2. 会话规则

1. 默认模式为 G-code 模式，LED0 点亮。
2. PS2 模式下 LED1 点亮，UART 接收数据会被丢弃，不能使用串口驱动机械臂。
3. 指令不区分大小写；参数顺序可变；空格可选。
4. 一行只能包含一条以 `G` 或 `M` 开头的命令。未知字母、重复参数、多个命令或 M 命令附带参数都会导致解析失败。
5. `G0/G1` 先进行整条路径的插补点校验。只有校验通过后，固件才返回 `ok` 并开始向运动队列持续下发轨迹帧；该回复不保证与发送命令处于同一时刻。
6. `G0/G1` 的 `ok` 表示命令已被接受，**不表示机械臂已经到达目标点**。上位机在收到该回复前，不应将该命令视为成功。
7. 上位机需要等待运动结束时，应在最后一条运动命令后发送 `M400`，并等待返回 `ok` 后再下发下一阶段任务。
8. 发送 `M400` 后，固件在运动完成前不会继续取出后续串口命令。因此上位机必须等待该命令的回复。

推荐的上位机交互顺序：

```text
TX  G1 X100 Y180 Z160 F2000\r\n
RX  ok\r\n
TX  M400\r\n
RX  ok\r\n
TX  M5\r\n
RX  M5OK\r\n
```

## 3. 命令定义

### 3.1 G0 / G1：笛卡尔直线运动

```text
G0 [X<mm>] [Y<mm>] [Z<mm>] [F<mm/min>]
G1 [X<mm>] [Y<mm>] [Z<mm>] [F<mm/min>]
```

当前固件中，`G0` 与 `G1` 的执行方式相同：都执行平滑分段的笛卡尔直线运动。
每条 `G0/G1` 至少应包含一个 `X`、`Y`、`Z` 或 `F` 参数。

| 参数 | 含义 | 规则 |
|---|---|---|
| `X` | 目标 X 坐标，mm | 可省略；省略时保持上一次规划位置 |
| `Y` | 目标 Y 坐标，mm | 可省略；省略时保持上一次规划位置 |
| `Z` | 目标 Z 坐标，mm | 可省略；省略时保持上一次规划位置 |
| `F` | 进给速度，mm/min | 可省略；默认值为 3000，最大值为 3000 |

示例：

```text
G1 X100 Y180 Z160 F2000
G1 Z120
G0 X0 Y185 Z240 F1500
```

成功回复：

```text
ok
```

目标点不可达、任一插补点不可达、超出关节软件限位、速度超限、规划队列超时或安全故障时，固件返回错误而不执行该运动。路径校验在主循环中分批进行，因此错误回复也可能在命令发送后的短暂延迟后出现。

### 3.2 M3 / M5：夹爪

```text
M3    ; 打开夹爪
M5    ; 关闭夹爪
```

回复分别为：

```text
M3OK
M5OK
```

### 3.3 M114：查询规划位置与关节反馈

```text
M114
```

典型回复：

```text
M114 PLAN X:0.00 Y:185.00 Z:240.00 J:0.00,0.00,0.00 ENC:0.00,0.00,0.00
```

字段含义：

- `PLAN`：最后一条已接受运动命令对应的规划坐标，不等同于实时末端测量坐标。
- `J`：由编码器电机端角度、减速比和 Home Pose 换算得到的 3 个关节角度，单位为度。
- `ENC`：3 路 AS5600 的连续电机端角度，单位为度。

### 3.4 M119：查询限位开关

```text
M119
```

典型回复：

```text
M119 M1:OPEN M2:TRIGGERED M3:OPEN
```

`TRIGGERED` 表示对应限位开关处于有效低电平；`OPEN` 表示未触发。

### 3.5 M400：等待运动完成

```text
M400
```

当运动引擎空闲且队列为空时返回：

```text
ok
```

如果执行期间发生安全故障，返回：

```text
error: M400 aborted by safety fault
```

### 3.6 M999：故障恢复

```text
M999
```

该命令只在运动安全故障已锁定时有效。固件会依次执行：关闭限位监控、回零、重建 Home Pose 规划状态、重新校准编码器、清除故障并重新开启限位监控。该过程采用非阻塞回零状态机；回零期间收到的串口数据会被丢弃，避免恢复完成后误执行旧命令。

接受恢复请求后，先回复：

```text
M999: homing started
```

完成回零、规划器重置和编码器校准后，回复：

成功回复：

```text
M999OK
```

若当前没有安全故障，固件返回：

```text
error: M999 requires a safety fault
```

## 4. 错误与恢复

| 回复 | 原因与处理 |
|---|---|
| `error: Parse failed!` | 命令格式不符合语法；检查命令、参数和 CRLF。 |
| `error: incomplete command; CRLF required` | 接收超时前未收到完整 CRLF；发送真实结束符。 |
| `error: UART RX overflow; command discarded` | 上位机发送过快或一行过长；降低发送速率并等待回复。 |
| `error: command rejected (-7)` | 常见于速度或目标范围超限。 |
| `error: motion safety stop (...); re-home required` | 限位、编码器通信、软件限位或规划超时触发；排除原因后发送 `M999`。 |

上位机在通信超时后不应盲目重复运动命令。应先发送 `M114` 或重新建立已知状态；若已进入安全故障，按 `M999` 流程恢复。
