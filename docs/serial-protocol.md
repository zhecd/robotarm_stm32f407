# UART G-code 串口协议

本文档定义 `robotarm_stm32f407` 与上位机之间的 UART 文本协议。上位机应以本文档为唯一依据，不应通过诊断日志推测命令是否完成。

## 1. 通信参数

| 项目 | 参数 |
|---|---|
| 接口 | UART1：PA9 为 TX，PA10 为 RX |
| 波特率 | 115200 bit/s |
| 格式 | 8N1，无校验，1 个停止位 |
| 电平 | 3.3 V TTL；连接 USB 时需要 USB-TTL 转换器 |
| 接收 | DMA 循环缓冲区 |
| 发送 | 非阻塞中断发送队列 |
| 最大命令行 | 255 个可用字符 |
| 行结束符 | CRLF，即 `0x0D 0x0A` |

命令必须使用文本模式发送，且每一行的末尾必须发送真实的回车和换行字节。文本中的 `\\r\\n` 四个可见字符并不等同于结束符。

## 2. 会话规则

### 2.1 输出分类

串口输出分为两类。

- 协议回复不以 `#` 开头，例如 `ok`、`queued`、`M3OK`、`M5OK`、`M114 ...`、`M119 ...` 与 `error: ...`。上位机只能依据这些回复判断命令结果。
- 诊断日志统一以 `# ` 开头，例如 `# ENC ...`、`# [Homing] ...` 与 `# WARN: ...`。诊断日志只用于观察和调试，不表示命令成功、失败或运动完成。

所有输出均以 CRLF 结束。上位机应按完整行处理接收数据，不应把一次串口读取视为一条完整回复。

### 2.2 模式与语法

1. 默认工作在 G-code 模式，LED0 点亮。
2. PS2 模式下 LED1 点亮，固件会丢弃 UART 接收数据，串口不能驱动机械臂。
3. 指令不区分大小写；参数顺序可变；参数之间可使用空格，也可以省略空格。
4. 每行只能包含一条以 `G` 或 `M` 开头的命令。未知字母、重复参数、多个命令或为 M 命令附加参数都会导致解析失败。
5. 自动闭环复位正在执行时，新的 `G0/G1` 会被拒绝，返回 `error: command rejected (-8)`。等待复位结束后再发送运动命令。

### 2.3 运动队列与回复

`G0/G1` 使用先进先出（FIFO）坐标队列。队列可保存 **16 个尚未执行的路径点**，因此视觉上位机可以连续发送一段离散轨迹，固件会按接收顺序执行，不会主动省略中间点。

- `ok`：该移动命令已完成路径校验，并已交给运动规划器。它不表示机械臂已经到达目标点。
- `queued`：该移动命令已写入坐标队列，稍后将按顺序校验和执行。它同样不表示已到达目标点。
- `error: command rejected (-5)`：坐标队列已满。该命令没有被接收；上位机应等待队列消耗，或发送 `M400` 等待当前运动完成后再重试。
- `error: queued command rejected (<错误码>); queue cleared`：某个已排队的点在真正开始校验时失败。由于后续点的参考位置不再可靠，剩余待执行路径点会被清空。上位机应读取 `M114`，重新建立已知状态后再发送轨迹。

缺省的 `X/Y/Z/F` 参数继承前一个**已接受**路径点的值，而非当前编码器的瞬时位置。因此，连续轨迹应以 `ok` 或 `queued` 为接收确认；收到错误后，不应假定该行已经改变后续坐标的参考值。

需要等待一段轨迹全部结束时，在最后一个运动命令后发送 `M400`。`M400` 会同时等待：正在校验的命令、运动规划器、步进运动队列以及坐标 FIFO 均为空。

推荐的交互顺序如下：

```text
TX  G1 X100 Y180 Z160 F2000\r\n
RX  ok\r\n
TX  G1 X120 Y180 Z160\r\n
RX  queued\r\n
TX  M400\r\n
RX  ok\r\n
TX  M5\r\n
RX  M5OK\r\n
```

上位机可以在收到 `ok` 或 `queued` 后发送下一条轨迹命令；如果任务需要确认末端已经静止，必须再等待 `M400` 返回 `ok`。

## 3. 命令定义

### 3.1 G0 / G1：笛卡尔直线运动

```text
G0 [X<mm>] [Y<mm>] [Z<mm>] [F<mm/min>]
G1 [X<mm>] [Y<mm>] [Z<mm>] [F<mm/min>]
```

当前固件中，`G0` 与 `G1` 的执行方式相同，均为分段插补的笛卡尔直线运动。每条 `G0/G1` 至少应包含一个 `X`、`Y`、`Z` 或 `F` 参数。

| 参数 | 含义 | 规则 |
|---|---|---|
| `X` | 目标 X 坐标，mm | 可省略；省略时保持前一个已接受路径点的 X |
| `Y` | 目标 Y 坐标，mm | 可省略；省略时保持前一个已接受路径点的 Y |
| `Z` | 目标 Z 坐标，mm | 可省略；省略时保持前一个已接受路径点的 Z |
| `F` | 进给速度，mm/min | 可省略；默认值为 3000，最大值为 3000 |

示例：

```text
G1 X100 Y180 Z160 F2000
G1 Z120
G0 X0 Y185 Z240 F1500
```

首个空闲运动命令需要在主循环中完成整条路径的插补点校验，因此 `ok` 或错误回复可能在发送后短暂延迟出现。后续排队命令先返回 `queued`，等轮到执行时才进行路径校验。目标点不可达、任一插补点不可达、超出关节软件限位、速度超限、规划超时或安全故障时，固件会返回错误而不执行该移动。

### 3.2 M3 / M5：夹爪

```text
M3    ; 打开夹爪
M5    ; 关闭夹爪
```

对应回复为：

```text
M3OK
M5OK
```

### 3.3 M114：查询规划位置与关节反馈

```text
M114
```

典型回复：

```text
M114 PLAN X:0.00 Y:185.00 Z:240.00 J:0.00,0.00,0.00 ENC:0.00,0.00,0.00
```

- `PLAN` 是最后一个已被规划器接受的笛卡尔位置，不是实时测得的末端坐标。
- `J` 是按编码器电机端角度、减速比和 Home Pose 换算的关节角度，单位为度。
- `ENC` 是 3 路 AS5600 的连续电机端角度，单位为度。

### 3.4 M119：查询限位开关

```text
M119
```

典型回复：

```text
M119 M1:OPEN M2:TRIGGERED M3:OPEN
```

`TRIGGERED` 表示对应限位开关为有效低电平；`OPEN` 表示未触发。

### 3.5 M400：等待全部运动完成

```text
M400
```

当运动引擎空闲、规划器空闲且坐标队列为空时，回复：

```text
ok
```

如果等待期间发生安全故障，回复：

```text
error: M400 aborted by safety fault
```

### 3.6 M999：故障恢复与回零

```text
M999
```

该命令只在安全故障已锁定时有效。固件会执行回零、重建 Home Pose 规划状态、重新校准编码器并恢复限位监控。回零采用非阻塞状态机；回零期间接收的串口数据会被丢弃，避免恢复结束后误执行旧命令。

接受恢复请求后首先回复：

```text
M999: homing started
```

恢复成功后回复：

```text
M999OK
```

当前没有安全故障时，回复：

```text
error: M999 requires a safety fault
```

## 4. 常见错误与处理

| 回复 | 原因与处理 |
|---|---|
| `error: Parse failed!` | 命令不符合语法；检查命令、参数和 CRLF。 |
| `error: incomplete command; CRLF required` | 接收超时前未收到完整 CRLF；发送真实结束符。 |
| `error: UART RX overflow; command discarded` | 发送过快或单行过长；降低发送速率并等待协议回复。 |
| `error: command rejected (-5)` | 坐标 FIFO 已满；等待 `M400` 或待队列消耗后重试。 |
| `error: command rejected (-7)` | 常见于速度或目标范围超限。 |
| `error: command rejected (-8)` | 运动系统忙碌，例如自动闭环复位正在执行。 |
| `error: queued command rejected (...)` | 已入队路径点校验失败；队列已经清空，读取 `M114` 后重新发送轨迹。 |
| `error: motion safety stop (...); re-home required` | 限位、编码器通信、实际关节软限位、规划超时或闭环补偿发散触发；排除原因后发送 `M999`。 |

上位机在通信超时后不应盲目重复运动命令。应先发送 `M114` 重新确认规划状态；若已进入安全故障，则按 `M999` 流程恢复。
